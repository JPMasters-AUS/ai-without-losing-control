# Issues Register

> [!NOTE]
> An issues register tracks unresolved questions, disagreements, risks, ambiguities, and non-blocking concerns. Not every issue is a blocker. Many can be accepted, tracked, and resolved later. What matters is that they are visible and owned, not hidden in chat history. This register reflects sound project practice: at the endorsement stage of any well-run project, it is normal for reviewers to identify minor issues that do not prevent progress but should be tracked.

## How to Use This Register

Add a new row for each issue identified. Update the status as issues are resolved or accepted. Link issues to decisions and requirements where relevant.

Issue types:

| Type | Meaning |
|---|---|
| Unresolved Question | A question that has not yet been answered. |
| AI Disagreement | Two AI tools gave materially different outputs for the same question. |
| Assumption | Something taken to be true without explicit confirmation. |
| Ambiguity | A requirement, decision, or constraint that is unclear or open to interpretation. |
| Risk | A potential problem that has not yet occurred but should be monitored. |
| Non-Blocking Concern | A concern that does not prevent progress but should be tracked. |
| Dependency Gap | A required input or decision from another party has not yet been received. |
